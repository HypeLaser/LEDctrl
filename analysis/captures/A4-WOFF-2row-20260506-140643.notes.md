## Wire summary

NMG UDP len 233. Payload **193 B**, identical length to A4-WON.

## NMG body — diff vs A4-WON

Single byte changes: `62 08` → `61 08`. All other bytes identical, including text run `LINE ONE 0d LINE TWO 0d 04`.

So when text fits within row width and rows are explicit, **wrap setting changes ONE byte (the wrap selector)** with NO impact on row layout. Wrap is a "what to do if a row overflows" instruction, not a layout directive.

## CRC validate

CRC carried in validate packet (56B, opcode 0x02:0x0e), last 4 bytes `[CRC_BE16][00 00]`.

A4-WON CRC bytes: `ec 0b` (BE16) = **0xEC0B**
A4-WOFF CRC bytes: `7e 14` (BE16) = **0x7E14**

Algorithm: CRC-16/X-25 (poly 0x1021, init 0xFFFF, refin/refout, xorout 0xFFFF)
Domain: NMG body from `01 5a 30 30` magic, claim_len bytes (excludes leading `01 00 01 00` outer-descriptor prefix).

Verified across all 5 A-set captures via `/tmp/verify_a_set.py` + `/tmp/verify_crc.py`. Wire stores CRC big-endian (NOT LE). Earlier note revisions labelled bytes as LE u16 (0x0BEC / 0x147E) — that was wrong byte-order interpretation.

Different between WON/WOFF: X25 of NMG body changed by 1 byte → drastic CRC drift, expected for X25.

## CRITICAL slot1 finding

**A4-WON and A4-WOFF have identical slot1 = 22240** despite different NMG content (CRC differs).

Slot1 is NOT derived from the NMG body bytes. Slot1 is an **Editor-side computed value** based on the "logical message" (rows, text, speed, pause, In/Out, etc.) — independent of how that message gets encoded for the wire.

A4-WON == A4-WOFF: **slot1 ignores wrap when wrap doesn't change layout** (text fits row).

A2-WON ≠ A2-WOFF (8224 vs 29300): **wrap changes layout dramatically (paginated vs scroll), slot1 changes accordingly**.

So slot1 reflects display duration of the on-sign rendering, computed from:
- Number of pages/scrolls
- Per-page dwell time (function of Pause + In/Out)
- Per-scroll length × speed
