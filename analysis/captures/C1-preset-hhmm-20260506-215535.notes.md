## NMG body diff vs A1 (static "HI")

Same 178-byte template, only the text-content region differs:

```
A1: ... 07 30 | 48 49           | 0d 04 ...    "HI" literal
C1: ... 07 30 | 0b 2f           | 0d 04 ...    clock token + format
                ^^ ^^
                |  +-- format selector 0x2f = HH:MM 24h preset
                +----- 0x0b = CLOCK / live-time token marker
```

## Discoveries

- **`0x0b`** = clock-insert token marker. Sits inline inside a text block (after `07 30` text-begin).
- **`0x2f`** = format selector for **HH:MM 24-hour, no seconds, no AM/PM**.
- Pattern is `0b <fmt>`, 2 bytes total. Token replaces literal characters in the text stream.
- Live-token can co-exist with literal characters in the same text block — composability confirmed structurally (not yet captured mixed).

## Side-finding — In/Out effect code drift

A1 ("HI", "Jump out"/"Jump out") emitted `0a 49 34 0a 4f 30` (I4 / O0).
C1 (clock token, "Jump out"/"Jump out") emitted `0a 49 30 0a 4f 30` (I0 / O0).

Same UI label "Jump out" → different In code (`4` vs `0`).
Hypothesis: editor session state, or context (text vs live-token) selects different default.
Doesn't block clock decode but flag for K-IN-* sweep later.

## Header bytes vs K-MTIME captures

K-MTIME-1/2 (length 221, NOT clock captures — plain "HI" with czSetTime context) show
prefix `1b 30 62 18 01 09 08 31 0e 32 30 30 30 31`.
C1 (length 218) shows `1b 30 62 08 31 0e 32 30 30 30 32` — **drops `18 01 09`**.

C1 NMG length is 178 (0xb2), K-MTIME is 181 (0xb5). 3-byte delta = the missing `18 01 09`.
Reason: K-MTIME captures used different editor state (likely "fitted init" sub-mode set).

## Open questions for C2..C10

- Is `0x2f` numerically meaningful (e.g. format index 47), or is it ASCII `'/'` repurposed as enum tag?
- Does Define Time (parameterised) emit same `0b 2f` or new token?
- AM/PM preset → `0b XX` with different XX?
- HH:MM:SS via Define Time Dis-Sec ON → still 2-byte `0b XX`, or longer?
- Three-separate-inserts (Hour + ":" + Min + ":" + Sec) → three `0b YY` tokens with literal `:` between?
- Date / day-of-week → new top-level marker (not `0b`) or extended `0b` family?
- Countdown / countup → likely need parameter bytes (target date/time) → longer token form.
