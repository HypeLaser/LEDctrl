## Editor input

Identical to C1 except clock preset: **HH:MM AM/PM** instead of HH:MM 24h.

## NMG body diff vs C1

```
C1: ... 07 30 0b 2f 0d 04 ...    HH:MM 24h
C2: ... 07 30 0b 30 0d 04 ...    HH:MM AM/PM
                ^^
                +-- format byte changed 0x2f → 0x30
```

Single-byte delta. NMG length unchanged (178). All other body bytes byte-identical.

## Findings

- **Format selectors are a sequential enum**, not ASCII-meaningful.
  - `0x2f` = HH:MM 24-hour
  - `0x30` = HH:MM AM/PM (12-hour)
- Each preset = next byte in the table. Strongly implies adjacent values for adjacent presets.
- Colon-blink behaviour is **baked into the format**, not a separate parameter.
  Preset 0x2f static colon (assumed — C1 didn't report flashing).
  Preset 0x30 flashing colon.
  Confirms why "Colon Static" toggle exists only on Define Time, not presets.
