### Critical finding

**Slot1 = 3000.** Not NMG length, not pixel count, not char count. Likely a **duration in ms** (3.0 s) ≈ pause(2s) + in/out transition slack. Confirms suspicion: SQ slot1 is a TIMING/DWELL value, not a length. Marquee gap bug = code currently emits NMG length here.

Need 2nd capture diffing one timing knob (Pause or Speed) to lock units.

### Token observations vs prior protocol-findings

- `1b 30` (auto-typeset value `0`) appears BEFORE `62 08`. Earlier docs said fitted init = `18 01 09` — not present here. May be vendor-version skew or my prior decode misread.
- `0a 49 34` and `0a 4f 30` confirm In/Out = `0a I<code>` / `0a O<code>` literal. **Jump out** maps to In `4`, Out `0`. Two distinct codes for the same UI label "Jump out" — interesting. Confirm in K-IN-* captures.
- `0f 33` = speed `'3'` (ascii). User-selected "medium" → vendor wrote `3`. Reveals speed encoding scale.
- `1c 31` = red ✓
- `1d 30` = bg black ✓
- `1a 31` = size 1
- Padding after footer: ASCII vendor signature `NoteNmg file version:v3.99` followed by spaces to 178.
