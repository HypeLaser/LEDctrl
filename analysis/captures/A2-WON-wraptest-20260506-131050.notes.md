### Critical wrap finding

- Editor split 24-char input into TWO rows at the space between "WIDE" and "LINE".
- Single inline `0d` (row sep) injected, with leading `0x20` space on the second row preserved.
- **Wrap is encoded inline in the NMG body, not in a header bit.** Wrap ON ≠ a separate flag — Editor performs the split client-side and emits `0d` separators where it would line-break.
- Pad signature length growth: NMG body still ends in `04`, then identical vendor-pad signature, total padded to 201 instead of 178.

### Slot1 update — NOT a fixed timing

A1: NMG=178, slot1=3000.
A2-WON: NMG=201, slot1=8224.

Diff is too large to be linear in length (+23 bytes → +5224). Not 2× rows × 3000 either (=6000).

Hypotheses:
1. **CRC16 of NMG body** — needs CRC trial against multiple algos
2. **Per-row aggregate display duration** including paginated dwell
3. **sigmaX25 partial digest** — protocol-findings notes existing sigmaX25 CRC use elsewhere

Action: **try CRC16 first** on NMG body once we have 3rd sample. If CRC, will fall out immediately. If not, vary Pause/Speed knobs to test timing hypothesis.

### Timestamp bytes

A1: `26 20 05 11 01 01 01 01`
A2: `26 20 05 13 01 01 01 01`

Byte index 3 incremented `11 → 13` (skipped 12). Possibly a session/file version counter that bumps on each new Editor save. Not load-bearing for replay.
