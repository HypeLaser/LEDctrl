## What Works

### Three Render Modes (locked)
- **Pages (`.fitted`)** — mode 'b', auto-paginate, all rows on one page, Jump Out effect.
- **Marquee (`.marquee`)** — mode 'a', continuous scroll, 20-space spacer between rows, min 22-char content (forces scroll).
- **Effects (`.slides`)** — mode 'a', each row = own page, `0x0d` separator. Per-row inline effect-change bytes (`0x0a 0x49 <inCode>` / `0x0a 0x4f <outCode>`).

### Per-Row Effects Override (NEW this session)
- `RowEffectOverride` UI struct (DataModels.swift) — id, enabled, inMode, outMode.
- `RescueModel.rowEffects` `@Published` array, length-synced to canvas line count via `syncRowEffectsToCanvasLines()`.
- `wireRowEffects()` builds wire payload: middle rows (not first/last) emit IN-only override; first/last emit IN+OUT.
- `SigmaTextOptions.RowEffect` — both `inEffectCode` and `outEffectCode` are now `UInt8?` so each axis can be omitted.
- `EffectsRenderer.renderSlidesBytes(rows:defaultFont:perRowEffects:)` — emits each axis independently when set; nil falls back to NMG header global In/Out.
- UI: new "Per-Row Effects" GroupBox in MessagesView controlColumn, only visible when `messageMode == .slides`. Each row gets toggle + IN picker; OUT picker only shown for first and last row.

### UI Mode Affordances
- In/Out/Speed pickers greyed out when mode != Effects (Pages and Marquee don't use them).
- Mode-specific helper text below Motion GroupBox.

### Scrolling Data Sources (RSS Scheduler)
User intent: scrape headlines at HH:59, scroll on HH:00 every hour. Sigma Editor 3.99 had a built-in scheduler for this.

Proposed approach (NOT YET BUILT):
- `ScheduleSource` model: URL + cadence + canvas slot.
- Background `Task` polls minute boundaries; at HH:59 fetches feed, parses titles, stores pending message; at HH:00 swaps live message to scrolling marquee for N seconds, reverts.
